# Coding Guidelines

## Tech Stack

- Plain JavaScript (no TypeScript). Do not introduce TypeScript or migrate existing files.
- React 19 + Vite for the `alysha-calendar` app.
- ESLint is the linter — keep it passing, but don't tighten or change rules without asking.

## Testing

- No tests required. This is a personal project; skip unit, integration, and component tests unless explicitly requested.

## Scope Limits

- Do not install new npm packages without asking first. Describe what the package does and why it's needed, then wait for approval.

## Output Format

- Before making any code change, describe the plan in plain English (what you'll change and why).
- Apply changes only after the user confirms the plan.
- Exception: trivial one-liners (e.g., fixing a typo) can be applied directly with a brief note.
